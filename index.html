Accessibility is the practice of ensuring that websites can be used by everyone regardless of whether they have a visual, physical, auditory or cognitive disability. The accessibility guidelines are structured around four key principles which help create websites that are perceivable, operable, understandable and robust.

Accessibility is important because, by making a conscious effort to address issues that affect users of different abilities, you maximise the overall number of people who can use the site :)

The checklist was created using the Web Content Accessibility Guidelines (WCAG) 2.0. to meet level AA. Links below each guideline will direct you to the WCAG website to find out more info.

Principle 1: Perceivable

This principle focuses on ensuring the user can operate, interact and navigate the site using an assistive device.

1.1: Text Alternatives.

For all non-text content that has a critical purpose such as to provide vital information or respond to user input, provide text alternatives.

  • 1.1.1.
    For non text content (images, audio, video) have you provided text alternatives and named all of your buttons (e.g ‘search or submit’)?
    - Show exceptions
    - How to pass criteria
    - About this criteria

1.1.1 Guidline Exceptions

+ If the non-text context is a control or accepts user input , then have you provided a name that describes it purpose?

+ If non-text content is time-based media, then does it have a name that describes its purpose?

+ If the non-text content is a test or exercise that would be invalid if presented in text, does the text alternative at least provide descriptive identification of the non-text content?

+ If it intended to create a specific sensory experience then does the text alternative at least provide descriptive identification of the non-text content?

+ If non-text is purely decorative then is it implemented in a way that can be ignored by assistive technology?


1.2: Accessibility for time-based media content.

For time based media content, include a variety of methods for presenting the equivalent information.

  • 1.2.1.
    For audio-only content have you provided a text transcript? For video content have provided subtitles?
    - How to pass criteria
    - About this criteria

1.3: Adaptable content.

Maintain separation of the presentation from the information, functionality and structure of the site so that the content does not lose its meaning when its presentation is changed or the visuals are removed.

  • 1.3.1.
    Have you used ARIA (Accessible Rich Internet Applications) appropriately in your markup by doing the following:

    + Have you used the role attribute to define what an element is or does (e.g. article, banner, main)?
    + Have you used properties to define the property or meaning of elements. For example, would it be possible for a screen reader to identify fields where the input is required?
    + Have you used states to identify the current condition of an element that changes based on user interaction or some dynamic variable?

    - How to pass criteria
    - About this criteria

  • 1.3.2.
    Is content ordered in a meaningful sequence and maintains its meaning when linearised, such as when style sheets are disabled? (Can a correct reading sequence be programatically determined.)?

    - How to pass criteria
    - About this criteria

  • 1.3.3
    Have you avoided creating content that relies solely on its colour, shape, location or sound as its sole method of conveying its meaning?

    - How to pass criteria
    - About this criteria

1.4: Distinguishable content.

Ensure it is easy for users to perceive the content.

  • 1.4.1.
    Have you avoided using colour as the only visual means of conveying information, indicating an action, prompting a response or distinguishing a visual element?

    - How to pass criteria
    - About this criteria

  • 1.4.2.
    Have you provided audio control so that for any audio with auto-play (that plays for more than 3 seconds) users are able to pause and stop it and can control the volume of it independently to controlling the volume of there computer/device (so it does not inter with screen reader volume).

    - How to pass criteria
    - About this criteria

  • 1.4.3.
    Have you tested the visual presentation of text and images to make sure it has a contrast ratio of at least 4.5:1. Unless it is large text which can have a contrast ratio of at least 3:1, text/images of text that is purely decoration or text that is part of a logo.

    - How to pass criteria
    - About this criteria

  • 1.4.4.
    Can text be resized up to 200 percent without loss of content or functionality without assistive technology? (Captions and images of text are exempt from this)

    - How to pass criteria
    - About this criteria

  • 1.4.5.
    Wherever possible (for example when the design of the text is not essential to the information being conveyed) have you used text to convey information rather than an image of text?

    - How to pass criteria
    - About this criteria

Principle 2: Operable

This principle focuses on ensuring the user can operate the site and interact with the user interface components and navigation.

2.1 Ensure the user can operate all site functionality using the keyboard.

  • 2.1.1
    Can the entire site including forms, be operated using a keyboard?
    - How to pass criteria
    - About this criteria

  • 2.1.2
    Have you tested that the there are no keyboard traps (where using the keyboard causes the focus to get trapped moving within subsections on the site)?
    - How to pass criteria
    - About this criteria

2.2 Ensure users have enough time to read content and complete tasks such as filling out a form before it expires or content scrolls.

  • 2.2.1
    Are timings adjustable so that for each time limit, the user can either turn off the timer, adjust it before they encounter it or extend it (unless it is one of the exceptions)?
    - Show exceptions
    - How to pass criteria
    - About this criteria

    • 2.2.1 Guideline Exceptions

    + Unless there is a real time exception and there is no alternative or if adjusting the time would invalidate the activity or if the time limit is longer than 20 hours.

  • 2.2.2
    For moving, blinking or scrolling information (when these actions are not essential) that starts automatically or lasts longer than five seconds is there a mechanism for the user to pause,stop or hide?

    For auto-updating information that starts automatically or is presented in parallel with other content, can the user pause, stop,hide or control the frequency pause, stop, or hide it or to control the frequency of the update?
    - How to pass criteria
    - About this criteria

2.3 Avoid using content that could cause seizures.

  • 2.3.1
    Have you avoided including anything on the site that contains anything that flashes more than 3 times per second?
    - How to pass criteria
    - About this criteria

2.4 Help users navigate and find content.

  • 2.4.1
    Have you ensured that the main content is easy to get by providing mechanism to bypass blocks of repeated content.

    - About this criteria

  • 2.4.2
    Are all pages titled clearly to reflect the page’s topic with clear descriptive headings for the sites and individual components (e.g. form with “required” labels) so information within the page can be easily located?
    - How to pass criteria
    - About this criteria

  • 2.4.3
    When navigating through content using the keyboard, is the order that the information is encountered (the focus order) consistent with the meaning of the content?
    - How to pass criteria
    - About this criteria

  • 2.4.4
    Can purpose of each link be determined by the link text alone or from the link text together with its programmatically determined link context (such as the text that is in the same paragraph).
    - How to pass criteria
    - About this criteria

  • 2.4.5
    Have you provided more than one method (together with the navigation menu) to navigate to other pages on the site (such as a search box and site map)?
    - How to pass criteria
    - About this criteria

  • 2.4.6
    Do all headings and labels clearly describe the topic or purpose of the content or page?
    - How to pass criteria
    - About this criteria

  • 2.4.7
    While using the keyboard is it clearly visible where on the site the keyboard currently has focus?

Principle 3: Understandable

This principle focuses on ensuring the user is able to process and understand the information even if they are.

3.1 Ensure the language of the site can be programmatically determined so the content can be displayed correctly by technology such as screen reader.

  • 3.1.1
    For each web page has the human language of the site been defined in the HTML ( lang=“en” ) so that is can be programmatically determined?
    - How to pass criteria
    - About this criteria

  • 3.1.2
    For each passage within the content, is it possible for the human language to be programmatically determined?
    - How to pass criteria
    - About this criteria

3.2 Content appears and operates in predictable ways.

  • 3.2.1
    3.2.2 For navigational mechanisms (such as the nav menu), have you maintained consistency across the pages?

  • 3.2.2
    Have you avoided creating any unexpected un-warned changes of context arising on input on the user interface components?
    - How to pass criteria
    - About this criteria

  • 3.2.3
    Have you maintained consistency in choice of navigational mechanisms (such as the nav menu) across the pages?
    - How to pass criteria
    - About this criteria
